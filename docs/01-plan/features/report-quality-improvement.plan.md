# Plan: report-quality-improvement

## Executive Summary

| 관점 | 내용 |
|------|------|
| **Problem** | 현재 주간보고서가 업무 로그처럼 30개+ 세부항목을 나열하여, 성과와 임팩트가 드러나지 않음 |
| **Solution** | 프롬프트 작성 규칙을 개선하여 카테고리별 핵심성과 + 정량지표 + 달성률 구조로 변경 |
| **기능/UX 효과** | 보고서가 간결하면서도 깊이 있고, 수치로 성과가 명확히 보임 |
| **Core Value** | 보고 받는 사람이 30초 내에 한 주의 성과와 진척을 파악 가능 |

## 1. 배경 및 문제 정의

### 현재 상태
- `prompts/weekly-report-prompt.md`의 "이번 주 한일" 작성 규칙: "프로젝트/주제별 그룹핑 + 완료형 작성 + 중복 제거"
- AI가 Notion 일지를 단순 재배열하여 세부항목을 그대로 나열
- 지난주 보고서: 6개 카테고리, 30개+ 세부항목, 정량지표 0건

### 문제점
1. **과도한 세부 나열**: 카테고리당 5~8개 세부항목으로 업무 로그 수준
2. **정량 지표 부재**: "~완료", "~구현" 서술형만 사용, 수치 없음
3. **계획-실적 단절**: 이번 주 계획과 한일이 독립적으로 작성되어 달성률 파악 불가
4. **이슈 미활용**: 미팅 의사결정사항이 이슈 테이블에 반영되지 않음

## 2. 목표

| 목표 | 측정 기준 |
|------|----------|
| 간결화 | 카테고리당 핵심성과 1~2줄 (세부항목 없음) |
| 정량 지표 | 모든 카테고리에 수치 1개 이상 포함 |
| 계획-실적 연결 | 카테고리별 달성률 [N%] 표시 |
| 이슈 활용 | 미팅 의사결정사항을 이슈 테이블에 자동 정리 |

## 3. 수정 범위

### 수정 대상: `prompts/weekly-report-prompt.md`

| 섹션 | 현재 | 변경 |
|------|------|------|
| §3 이번 주 계획 | 프로젝트별 계획 항목 나열 | 프로젝트별 핵심 목표 2~4개, 측정 가능한 형태 |
| §4 이번 주 한일 | 프로젝트/주제별 그룹핑, 세부항목 나열 | 카테고리별 1~2줄 핵심성과 + 정량지표 + 달성률 |
| §5 차주 계획 | 후속작업 위주 제안 | 이번 주 미완료 항목 이월 + 신규 목표 |
| §6 이슈 정리 | 업무일지 이슈만 추출 | 미팅 의사결정사항 + 블로커 + 리스크 포함 |

### 수정하지 않는 것
- n8n 워크플로우 (HTML 템플릿 구조는 유지)
- webhook contract (필드 변경 없음)
- Confluence 페이지 테이블 구조

## 4. 구현 계획

1. `prompts/weekly-report-prompt.md` §3~§6 작성 규칙 업데이트
2. 출력 예시(before/after) 추가하여 AI가 정확히 따르도록 유도

## 5. 리스크

| 리스크 | 대응 |
|--------|------|
| Notion 일지에 수치 데이터가 없으면 AI가 정량화 불가 | AI가 항목 수를 세어 자동 수치화하도록 규칙 명시 |
| 과도한 압축으로 중요 정보 누락 | 카테고리별 핵심성과 형식을 예시로 고정 |
